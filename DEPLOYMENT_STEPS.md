## 🔧 Troubleshooting

### Error: "Cannot read properties of undefined (reading 'status')"
**Solution**: Make sure safe navigation operators are in place:
```javascript
$response?.body?.candidates?.[0]?.content?.parts?.[0]?.text
```

### Error: "Invalid API key provided"
**Solution**: 
1. Verify GEMINI_API_KEY is set in n8n environment
2. Verify key is valid at: https://console.cloud.google.com/
3. Ensure key has Generative Language API enabled

### Error: "API key quota exceeded"
**Solution**: 
1. Check API quotas in Google Cloud Console
2. Adjust workflow batch sizes if needed
3. Space out workflow executions

### Workflow doesn't trigger on schedule
**Solution**:
1. Verify cron expression is correct
2. Check n8n timezone settings
3. Manually trigger workflow to test logic

### Response parsing returns empty/null
**Solution**:
1. Check actual API response in n8n execution logs
2. Gemini response structure might be different
3. Adjust parsing path based on actual response

## ✨ What Changed Summary

### API Endpoints
```diff
- https://api.groq.com/openai/v1/chat/completions  ❌
- https://api.gemini.google.com/v1/models/...      ❌
+ https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-{flash|pro}:generateContent?key={{ $env.GEMINI_API_KEY }}  ✅
```

### Authentication
```diff
- headerAuth with Bearer token  ❌
+ queryAuth with API key parameter  ✅
```

### Request Format
```diff
- Old Groq format  ❌
+ Gemini standard: {"contents": [{"role": "user", "content": "prompt"}]}  ✅
```

### Response Parsing
```diff
- Unsafe: $response.body.candidates[0].content.parts[0].text  ❌
+ Safe: $response?.body?.candidates?.[0]?.content?.parts?.[0]?.text || fallback  ✅
```
