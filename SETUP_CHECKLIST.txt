🔧 SETUP OPTION 1: Cloudflare Tunnel (No Port Forwarding)
────────────────────────────────────────────────────────────
Estimated Time: 15 minutes
Benefits: No ISP port forwarding, DDoS protection, works anywhere

Steps:
  1. Install cloudflared: brew install cloudflare/cloudflare/cloudflared
  2. Run: cloudflared tunnel login
  3. Run: cloudflared tunnel create nexus
  4. Copy Tunnel Token to .env: CLOUDFLARE_TUNNEL_TOKEN=<paste-here>
  5. Start service: docker compose --profile cloudflare-tunnel up -d
  6. Add DNS CNAME records in Cloudflare Dashboard
  7. Test: curl https://app.nexus-io.duckdns.org

Detailed Guide: CLOUDFLARE_TUNNEL_SETUP.md

🔧 SETUP OPTION 2: SSH Key Authentication
──────────────────────────────────────────
Estimated Time: 15 minutes
Benefits: Immune to password brute force, unhackable phishing

Steps:
  1. Generate key: ssh-keygen -t ed25519 -f ~/.ssh/nexus_server
  2. Copy to server: ssh-copy-id -i ~/.ssh/nexus_server redbend@77.133.1.37
  3. Edit server /etc/ssh/sshd_config:
     - Set: PasswordAuthentication no
     - Set: PermitRootLogin no
     - Set: MaxAuthTries 3
  4. Restart SSH: sudo systemctl restart sshd
  5. Test with new terminal (keep old one open): ssh nexus

Detailed Guide: SSH_KEY_SETUP.md

🔧 SETUP OPTION 3: Fail2ban (Brute Force Protection)
──────────────────────────────────────────────────────
Estimated Time: 10 minutes
Benefits: Auto-ban IPs after failed logins, configurable thresholds

Steps:
  1. Start service: docker compose --profile fail2ban up -d fail2ban
  2. Create /etc/fail2ban/jail.local with config (see guide)
  3. Start service: sudo systemctl start fail2ban
  4. Enable on boot: sudo systemctl enable fail2ban
  5. Monitor: sudo fail2ban-client status sshd

Detailed Guide: FAIL2BAN_SETUP.md

SUPPORT & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: Can't delete account
  → Check browser console for errors
  → Verify you're logged in
  → See: app/api/user/route.ts

Problem: Admin clients page won't load
  → Check user.isAdmin = true
  → Verify authentication
  → See: app/admin/clients/page.tsx

Problem: SSH key not working
  → Check permissions: chmod 600 ~/.ssh/nexus_server
  → Verify key is in authorized_keys
  → See: SSH_KEY_SETUP.md → Troubleshooting

Problem: Cloudflare Tunnel not connecting
  → Check token in .env
  → Verify Docker service running
  → See: CLOUDFLARE_TUNNEL_SETUP.md → Troubleshooting
